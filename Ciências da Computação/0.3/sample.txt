ESPECIFICAÇÕES TÉCNICAS OBRIGATÓRIAS
- Login com autenticação via token
- Todos os cruds devem ser funcionais e serem enviados ou trazidos da api. Sendo eles: usuários,  serviços, categorias, prestadores e pedidos.
- Além dos usuários, prestadores, serviços, deve ser criado uma tabela para armazenar os pedidos feitos por cada cliente especificando para qual prestador, serviço e demais informações.
- O cadastro inicial antes de entrar no app deve conter apenas os dados: nome, email, telefone, senha, confirmação de senha.
- Deve existir a validação da senha caso a senha e confirmação de senha sejam diferentes.
- Na recuperação, deve ser gerado uma nova senha aleatória e enviada para o e-mail cadastrado. 
- Dentro na tela inicial a busca deve ser dinâmica, buscando automaticamente assim que a pessoa começar a digitar.
- O pedido de ser armazenado no banco de dados e listado na tela de histórico com o status de em andamento, concluído ou cancelado.
- As rotas da api devem ter a opção de serem testadas pelo software insominia ou outro de teste de api rest Full.
- No perfil do usuário deve ter a opção de alteração de foto do perfil além dos demais dados pessoais.
